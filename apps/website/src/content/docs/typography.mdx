## Text

The `Text` component is a polymorphic typography component that can be used to render any kind of text as any kind of element.

<LiveExample src='Text.main.jsx' truncate={false}>
  <AllExamples.TextMainExample client:load />
</LiveExample>

### Variant

The `variant` prop can be used to determine which type of typography size should be utilized for the content of the application. The available variants are:

- `"headline"`: can be treated as an HTML `<h1>` text.

- `"title"`: can be treated as an HTML `<h2>` text.

- `"subheading"`: can be treated as an HTML `<h3>` text.

- `"leading"`: can be treated as an HTML `<h4>` text.

- `"body"` (default): can be treated as an HTML `<p>` text.

- `"smaller"`: can be treated as an HTML `<smaller>` text.

</LiveExample>

### Muted

To achieve a muted or desaturated text effect, the `isMuted` prop can be passed into the component. Muted text helps differentiating less important information as well as enhancing the overall readability.

<LiveExample src='Text.muted.jsx' truncate={false}>
  <AllExamples.TextMutedExample client:load />
</LiveExample>

### Skeleton

To indicate that content is loading and will be available soon, use skeleton text by setting the `isSkeleton` prop on the component.

The `isSkeleton` prop can be passed along with `variant` to show indicate different text section being loaded. By default or with no `variant` being passed, the skeleton has the size of a `"body"` text.

<LiveExample src='Text.skeleton.jsx'>

</LiveExample>

### Polymorphic

The `Text` component supports the polymorphic `as`, allowing end users to render the text as a specific heading level while maintaining a desired size. This provides users the flexibility to control both the visual style and underlying HTML element used for rendering the texts.

## Label

The `Label` component is a fundamental element that provides descriptive text or captions for form inputs.

```jsx
<Label>This is a label.</Label>
```

`Label` can be used with `Input` components by setting the `htmlFor` attribute to match the `id` of the corresponding input element.

This association enhances accessibility by ensuring that screen readers and other assistive technologies can correctly identify the relationship between the label and the input field.
